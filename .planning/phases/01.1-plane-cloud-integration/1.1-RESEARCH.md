# Phase 1.1 Research: Plane Cloud Integration (Best Practices)

## Standard Stack
- **Library:** `requests` (Current) / `httpx` (Recommended for async).
- **Authentication:** OAuth 2.0 (Bot Token Flow) or User API Key (X-API-Key).
- **Verification Endpoint:** `/api/v1/users/me/` for connectivity checks.

## Architecture Patterns

### Authentication: Bot vs. User
- **User API Keys:** Simple to implement via `X-API-Key` header. Best for initial verification and internal tooling.
- **OAuth Bot Tokens:** Recommended for production bots. Features:
    - Independent identity (not tied to a specific user seat).
    - Support for `@mentions` in comments.
    - Granular permission scopes.
    - **Recommendation:** Start with API Key for Phase 1.1 verification, but plan for OAuth transition in Phase 4.

### Real-time Integration
- **Webhooks:** Plane Cloud supports webhooks for real-time events. 
    - **Pros:** Efficient, scalable, low latency.
    - **Cons:** Requires a public URL (difficult for local dev without ngrok).
- **Polling:** (Current strategy).
    - **Pros:** Works locally, no public ingress needed.
    - **Cons:** Higher latency, risks hitting rate limits (60/min).
- **Recommendation:** Use **Polling** for Milestone 2 development, but include a "Webhook Ingress" task in the future cloud deployment phase.

## Don't Hand-Roll
- **Signature Verification:** Use standard `hmac-sha256` for webhook verification if moving to webhooks.
- **Exponential Backoff:** Use libraries like `tenacity` or built-in Celery retry logic instead of custom sleep loops.

## Common Pitfalls
1. **Rate Limit Exhaustion:** Standard Plane Cloud API keys are limited to **60 requests per minute**. 
    - **Fix:** Track `X-RateLimit-Remaining` headers and implement client-side throttling.
2. **Double Pathing:** As discovered, `ADMIN_BASE_URL` logic in Plane can cause duplicated paths if prefix handling is inconsistent.
3. **Webhook Security:** Failing to verify `X-Plane-Signature` allows spoofing.

## Code Examples

### Authentication Header (Cloud API Key)
```python
headers = {
    "X-API-Key": os.getenv("PLANE_API_TOKEN"),
    "Content-Type": "application/json",
}
```

### Rate Limit Awareness (Pseudo-code)
```python
def check_rate_limit(response):
    remaining = int(response.headers.get("X-RateLimit-Remaining", 1))
    reset_time = int(response.headers.get("X-RateLimit-Reset", 0))
    if remaining == 0:
        wait_time = max(0, reset_time - time.time())
        time.sleep(wait_time)
```
